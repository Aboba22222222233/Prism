# 🔮 PRISM — Полный гайд по проекту

> **Prism** — это цифровая платформа школьного психолога. Учителя отслеживают эмоциональное состояние учеников, а ученики проходят чек-ины настроения. Искусственный интеллект анализирует данные и выявляет учеников в зоне риска.

## 🏗 Общая архитектура

```
┌──────────────────────────────────────────────────┐
│              ПОЛЬЗОВАТЕЛЬ (Браузер)               │
│    Landing → Login → Dashboard (Teacher/Student)  │
└──────────────────┬───────────────────────────────┘
                   │ HTTP / WebSocket
┌──────────────────▼───────────────────────────────┐
│               SUPABASE (Backend-as-a-Service)     │
│  ┌────────────┐  ┌──────────┐  ┌──────────────┐  │
│  │    Auth     │  │ Database │  │   Storage    │  │
│  │  (Google,   │  │ (Postgres│  │  (аватары,   │  │
│  │   email)    │  │  + RLS)  │  │   обложки)   │  │
│  └────────────┘  └──────────┘  └──────────────┘  │
│  ┌─────────────────────────────────────────────┐  │
│  │  Edge Functions (серверные функции)          │  │
│  │  chat-ai → Groq API → GPT-OSS / Llama 3.3  │  │
│  └─────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

**Принцип**: Клиентское React-приложение общается напрямую с Supabase. Для AI-запросов используются Edge Functions — серверные функции, которые безопасно хранят API-ключи и проксируют запросы к Groq API.

## 🛠 Технологический стек

| Категория | Технология | Зачем выбрали |
|-----------|-----------|---------------|
| **Фронтенд** | React 19 + TypeScript | Компонентный подход, строгая типизация |
| **Сборщик** | Vite 6 | Мгновенный запуск dev-сервера, быстрая сборка |
| **Стилизация** | Tailwind CSS (CDN) | Быстрая разработка UI без написания CSS-файлов |
| **Роутинг** | React Router DOM 7 | Навигация между страницами (SPA) |
| **Графики** | Recharts | Визуализация статистики настроения |
| **Иконки** | Lucide React | Единый стиль иконок по всему приложению |
| **Анимации** | Framer Motion | Плавные переходы и UI-анимации |
| **3D/Шейдеры** | Three.js + OGL | Визуальные эффекты на Landing-странице |
| **Backend** | Supabase | Auth, БД, Storage, Edge Functions — всё в одном |
| **AI** | Groq API (GPT-OSS 120B, Llama 3.3 70B) | Анализ эмоционального состояния, чат-бот |
| **Хостинг** | Vercel | Автоматический деплой из Git |
| **Экспорт** | ExcelJS + FileSaver | Экспорт данных в Excel-файлы |
| **Безопасность** | DOMPurify | Очистка HTML от XSS-атак |

## ⚙ Корневые файлы (конфигурация)

### `index.html`
**Что делает**: Главный HTML-файл. Всё приложение загружается внутрь `<div id="root">`.

**Инженерные решения**:
- Tailwind CSS загружается через CDN (`cdn.tailwindcss.com`) — не надо ничего устанавливать
- Шрифт Inter подключается из Google Fonts для современного вида
- Настроена кастомная тема Tailwind: фирменные цвета (slate-dark), шрифт Geist Sans
- Import Map позволяет использовать библиотеки напрямую в браузере без сборки

### `index.tsx`
**Что делает**: Точка входа JavaScript. Рендерит React-приложение в `<div id="root">`.

### `App.tsx`
**Что делает**: **Маршрутизатор** — определяет, какая страница открывается по какому URL.

| URL | Страница | Описание |
|-----|----------|----------|
| `/` | Landing | Главная (лендинг) |
| `/login` | Login | Авторизация |
| `/dashboard` | TeacherDashboard | Кабинет учителя |
| `/student-dashboard` | StudentDashboard | Кабинет ученика |
| `/checkin` | StudentCheckIn | Форма чек-ина |
| `/legal` | Legal | Юридические документы |
| `/parental-consent` | ParentalConsent | Согласие родителей |
| `/changelog` | Changelog | История обновлений |
| `/blog` | Blog | Блог |
| `/auth/mobile-callback` | MobileAuthCallback | OAuth-редирект для мобилки |

**Инженерные решения**:
- **Lazy Loading** (`React.lazy`) — каждая страница загружается только когда пользователь на неё переходит. Это сокращает начальную загрузку приложения
- **Suspense + LoadingFallback** — пока страница грузится, показывается анимированный спиннер

### `package.json`
**Что делает**: Список всех зависимостей (библиотек), скрипты для запуска.

**Команды**:
- `npm run dev` — запустить локально для разработки
- `npm run build` — собрать продакшн-версию
- `npm run preview` — посмотреть собранную версию

### `vite.config.ts`
**Что делает**: Конфигурация сборщика Vite.

**Инженерные решения**:
- **Manual Chunks** — тяжёлые библиотеки (recharts, three.js, react) вынесены в отдельные файлы. Браузер кеширует их отдельно, и при обновлении кода приложения не нужно перекачивать эти библиотеки
- Сервер слушает на `0.0.0.0:3000`, чтобы быть доступным по сети (для тестирования на телефоне)

### `vercel.json`
**Что делает**: Настройки хостинга Vercel.

**Инженерные решения**:
- **SPA Rewrite** — все URL перенаправляются на `index.html`. Это нужно, потому что роутинг на стороне клиента (React Router), а не сервера. Без этого обновление страницы на `/dashboard` давало бы ошибку 404

### `.env`
**Что делает**: Хранит секретные ключи (URL Supabase, анонимный ключ). Не загружается в Git.

## 📄 Страницы (src/pages/)

### `Landing.tsx` (610 строк)
**Что делает**: **Главная рекламная страница**. Первое, что видит посетитель.

**Содержит**:
- Навбар с бургер-меню
- Hero-секция с призывом к действию
- Секции "Для учителей" и "Для учеников" — описание функций
- Тарифные планы (Trial / Standard / Ultra) с ценами в тенге (₸)
- Секция отзывов (TestimonialsSection)
- Футер с навигацией

**Инженерные решения**:
- Используются кастомные карточки `EvervaultCard` — интерактивные карточки с шейдерным эффектом при наведении мыши
- Секция тарифов с переключателем месяц/год (PricingWithToggle)
- Цены форматируются с пробелами для читабельности (75 000₸)

---

### `Login.tsx` (362 строки)
**Что делает**: **Страница авторизации** — вход и регистрация.

**Функции**:
- Выбор роли: «Я учитель» / «Я ученик»
- Для учителей — проверка секретного кода перед регистрацией (защита от случайной регистрации как учитель)
- Вход по email + пароль
- Вход через Google (OAuth)
- После входа — автоматическое перенаправление на нужный дашборд в зависимости от роли

**Инженерные решения**:
- **Верификация кода учителя** — учитель должен ввести специальный код, который проверяется через серверную функцию Supabase (RPC `verify_teacher_code`). Это не даёт ученикам зарегистрироваться как учитель
- **Role-based redirect** — после входа система проверяет роль в таблице `profiles` и перенаправляет: учитель → `/dashboard`, ученик → `/student-dashboard`
- Фон — анимированный градиент (компонент ColorBends)

### `TeacherDashboard.tsx` (1559 строк) — Самый большой файл
**Что делает**: **Панель управления учителя** — главный рабочий инструмент.

**Функции**:
- Создание и управление классами
- Просмотр списка учеников в каждом классе
- KPI-карточки: средний балл настроения, кол-во чек-инов, активность
- Графики (Recharts): динамика настроения по дням
- **AI-анализ класса** — одной кнопкой получить AI-сводку по эмоциональному состоянию класса
- **AI Risk Assessment** — оценка каждого ученика по шкале 0–10 (нормальный / внимание / риск / критически)
- Создание заданий для учеников
- Создание событий (СОР, СОЧ, контрольные)
- Удаление учеников из класса
- Экспорт данных в Excel (ExcelJS) и CSV
- Удаление класса

**Инженерные решения**:
- **Row Level Security** — учитель видит данные только своих классов. Безопасность на уровне БД
- **KpiCard** — отдельный компонент для карточек с метриками
- **AI с fallback** — если AI-сервис недоступен, используется правило "среднее настроение ≤ 1.5 = риск"
- **Динамическая загрузка xlsx** — библиотека ExcelJS загружается через `import()` только в момент экспорта, чтобы не увеличивать начальный размер приложения

---

### `StudentDashboard.tsx` (1099 строк)
**Что делает**: **Панель ученика** — личный кабинет.

**Функции**:
- Просмотр всех классов, в которые записан ученик
- Вступление в новый класс по коду
- Выход из класса
- Статистика: средний балл настроения, серия чек-инов, уровень энергии
- Календарь настроения (MoodCalendar) — цветовая карта по дням
- История чек-инов с возможностью удаления
- **AI-анализ** — персональная сводка от AI по данным чек-инов
- AI-чатбот (MentorChat) — ученик может поговорить с AI-психологом
- Ресурсный хаб (ResourceHub) — встроенная библиотека ресурсов
- Задания от учителя
- Редактирование профиля (имя, аватар)

**Инженерные решения**:
- **Multi-class support** — ученик может быть записан в несколько классов и переключаться между ними
- **MoodCalendar** — визуализация настроения за месяц, каждый день окрашен в цвет от красного (плохо) до зелёного (отлично)
- **NavItem / StatCard** — переиспользуемые UI-компоненты для навигации и статистики

### `StudentCheckIn.tsx` (297 строк)
**Что делает**: **Форма чек-ина** — ученик заполняет данные о своём самочувствии.

**Шаги**:
1. Оценка настроения (1–5, эмодзи)
2. Выбор факторов влияния (категории: Школа, Семья, Общение, Здоровье, Личное)
3. Выбор эмоций (грусть, тревога, радость и т.д.)
4. Уровень энергии (1–10)
5. Часы сна
6. Комментарий (свободный текст)

**Инженерные решения**:
- **Пошаговый UI** — сложная форма разбита на простые шаги, чтобы не перегружать ученика
- Факторы разбиты по категориям (`categorizedFactors`) для удобства
- Данные отправляются в таблицу `checkins` с привязкой к `user_id` и `class_id`

### `ResourceHub.tsx` (444 строки)
**Что делает**: **Ресурсный хаб** — библиотека материалов для психологической поддержки.

**Содержит**:

**Инженерные решения**:
- **BreathingWidget** — самостоятельный виджет с фазами дыхания (вдох 4 сек → задержка 4 сек → выдох 4 сек), анимация кольца, кнопки play/pause/reset
- **DOMPurify** — весь контент очищается от потенциально опасного HTML (защита от XSS-атак)
- Компонент принимает `onClose` — может закрываться как модальное окно

## 🧩 Компоненты (src/components/)

### `ColorBends.tsx` (310 строк)
**Что делает**: **Анимированный градиентный фон** — визуальный эффект, создающий «живые» переливающиеся цвета.

**Инженерные решения**:
- Использует **WebGL-шейдеры** через Three.js — рендерится на видеокарте, не нагружает процессор
- Фрагментный шейдер (GLSL) создаёт процедурные перетекания цветов
- Реагирует на движение мыши (`mouseInfluence`, `parallax`) — создаёт эффект глубины
- Параметры: скорость, цвета, масштаб, частота, сила деформации — всё настраивается
- Используется как фон на странице логина

### `Prism.tsx` (458 строк)
**Что делает**: **3D-призма** — анимированный логотип / декоративный элемент в виде светящейся призмы.

**Инженерные решения**:
- Рендерится через **OGL** (минималистичная WebGL-библиотека, легче Three.js)
- Кастомный фрагментный шейдер создаёт эффект преломления света через призму
- Три режима анимации: вращение, реакция на курсор, 3D-вращение
- **Performance optimization**: `suspendWhenOffscreen` — останавливает рендер, когда элемент не видим

### `MentorChat.tsx` (255 строк)
**Что делает**: **AI-чатбот для учеников** — интерфейс чата с «психологом-наставником».

**Инженерные решения**:
- Выбор AI-модели: GPT-OSS 120B (Groq) или Llama 3.3 70B
- Системный промпт включает данные профиля ученика и статистику чек-инов — AI знает контекст
- Автоматическая прокрутка к последнему сообщению
- Анимация набора текста (typing indicator)

### `TeacherMentorChat.tsx` (245 строк)
**Что делает**: **AI-чатбот для учителей** — аналог MentorChat, но с контекстом класса.

**Инженерные решения**:
- Системный промпт включает: имя учителя, статистику класса (avg mood, risk count, active count), данные учеников, расписание событий

## 📚 Библиотеки (src/lib/)

### `supabase.ts` (7 строк)
**Что делает**: Создаёт и экспортирует клиент Supabase.

```
supabase = createClient(URL, ANON_KEY)
```

Этот клиент используется **везде** в приложении для:
- Авторизации (`supabase.auth.signInWithPassword`)
- Чтения/записи данных (`supabase.from('table').select()`)
- Вызова серверных функций (`supabase.functions.invoke()`)
- Загрузки файлов (`supabase.storage.upload()`)

**Инженерные решения**:
- Ключи берутся из переменных окружения (`import.meta.env.VITE_*`) — не хардкодятся в коде
- Единая точка инициализации — если нужно поменять настройки, правим только этот файл

### `gemini.ts` (134 строки)
**Что делает**: **Сервисный слой для AI** — все AI-вызовы проходят через этот файл.

**3 функции**:

1. **`getGeminiInsight(prompt, model)`** — одноразовый AI-запрос. Отправляет промпт, получает ответ. Используется для анализа класса и отдельных учеников.

2. **`getChatResponse(messages, model)`** — запрос с историей чата. Отправляет массив сообщений (системное + вся переписка), получает ответ. Используется в чат-ботах.

3. **`assessStudentRisk(studentData)`** — **оценка риска ученика**:
   - Формирует промпт «школьного психолога» с данными чек-инов ученика
   - Просит AI оценить риск по шкале 0–10 с расшифровкой
   - Парсит JSON-ответ AI
   - **Fallback**: если AI недоступен, считает по формуле (среднее настроение ≤ 1.5 → риск)
   - Возвращает: `{ isRisk, riskLevel, status, reason }`

**Инженерные решения**:
- Название файла «gemini» — историческое (изначально использовался Google Gemini), сейчас используется Groq API
- Все запросы проходят через Supabase Edge Function (`chat-ai`) — API-ключ не попадает к клиенту
- Обработка ошибок на каждом уровне + fallback для `assessStudentRisk`

### `utils.ts` (7 строк)
**Что делает**: Утилита `cn()` — объединяет CSS-классы с поддержкой Tailwind (через `clsx` + `tailwind-merge`).

## 🔒 Серверная часть — Supabase

### Edge Function: `chat-ai/index.ts` (76 строк)
**Что делает**: **Прокси к Groq API** — серверная функция, которая безопасно отправляет запросы к AI.

**Как работает**:
1. Получает запрос от клиента (messages, model, temperature)
2. Проверяет наличие Bearer-токена (авторизация)
3. Берёт `GROQ_API_KEY` из серверных переменных окружения
4. Отправляет запрос к `api.groq.com/openai/v1/chat/completions`
5. Возвращает ответ клиенту

**Инженерные решения**:
- **API-ключ на сервере** — ключ хранится в Supabase Secrets, не виден в клиентском коде
- **CORS-заголовки** — разрешают запросы с любого домена (для разработки)
- **Обработка OPTIONS** — для CORS Preflight-запросов
- Модель захардкожена как `openai/gpt-oss-120b` — клиент не может использовать произвольные модели (безопасность)

### Edge Function: `chat/index.ts` (11 строк)
**Что делает**: **Устаревший эндпоинт** — возвращает ошибку 410 (Gone). Был заменён на `chat-ai` после обнаружения, что API-ключи были в клиентском коде.

## 🗃 База данных — Миграции

### `supabase_schema.sql` (основная схема)
**Таблицы**:

| Таблица | Назначение |
|---------|-----------|
| `profiles` | Профили пользователей (id, email, имя, роль, аватар) |
| `checkins` | Чек-ины настроения (настроение 1–5, факторы, эмоции, сон, энергия) |
| `classes` | Классы учителей (название, уникальный код для вступления) |
| `class_enrollments` | Связь «ученик — класс» (many-to-many) |
| `tasks` | Задания от учителя для класса |
| `student_tasks` | Ответы учеников на задания |
| `class_events` | Календарные события (СОР, СОЧ, контрольные) |

**Row Level Security (RLS)** — ключевое инженерное решение:
- Ученик видит **только свои** чек-ины
- Учитель видит чек-ины **всех учеников** (для анализа)
- Учитель управляет **только своими** классами
- Ученик может вступить в класс и выйти, но не может менять чужие данные

### Остальные миграции:
| Файл | Что делает |
|------|-----------|
| `supabase_trigger.sql` | Триггер: при удалении записи из `class_enrollments` — очищает `class_id` в профиле ученика |
| `add_calendar.sql` | Добавляет таблицу `class_events` для календаря |
| `fix_db_error.sql` | Исправление ошибок в схеме |
| `fix_delete_permissions.sql` | Добавление прав на удаление (классы, задания, события) |
| `role_protection.sql` | Защита от смены роли — только сервер может менять роль пользователя |
| `verify_teacher_code.sql` | SQL-функция для проверки кода учителя при регистрации |

## 💡 Ключевые инженерные решения

### 1. Безопасность
- **RLS (Row Level Security)** — права доступа к данным на уровне базы данных, а не приложения. Даже если кто-то обойдёт фронтенд, база не отдаст чужие данные
- **API-ключи на сервере** — ключ Groq хранится в Supabase Edge Function, не в клиентском коде
- **Верификация роли учителя** — регистрация как учитель возможна только с секретным кодом
- **DOMPurify** — очистка пользовательского HTML от XSS-атак
- **CORS** — контроль, откуда могут приходить запросы

### 2. Производительность
- **Code Splitting** — каждая страница загружается отдельно (lazy loading)
- **Manual Chunks** — тяжёлые библиотеки (recharts, three.js) в отдельных файлах для кеширования
- **Dynamic Import** — ExcelJS загружается только при экспорте
- **WebGL на GPU** — шейдерные эффекты рендерятся на видеокарте
- **suspendWhenOffscreen** — 3D-элементы останавливаются, когда не видны

### 3. Архитектура
- **Supabase-as-Backend** — вся серверная часть без написания серверного кода (Auth, DB, Storage, Functions)
- **Edge Functions** — серверная логика максимально близко к пользователю (CDN)
- **SPA (Single Page Application)** — навигация без перезагрузки страницы
- **Компонентный подход** — UI разбит на переиспользуемые компоненты

### 4. AI-интеграция
- **Двухуровневая AI-система**: анализ данных + интерактивный чатбот
- **Risk Assessment** — автоматическая оценка рисков с промптом «школьного психолога»
- **Fallback** — если AI недоступен, система продолжает работать с rule-based оценкой
- **Мультимодельность** — поддержка нескольких AI-моделей (GPT-OSS, Llama)

### 5. UX/Дизайн
- **Пошаговый чек-ин** — сложная форма разбита на простые шаги с эмодзи
- **Дыхательное упражнение** — встроенный инструмент самопомощи
- **Mood Calendar** — визуальная карта настроения за месяц
- **Dark theme** — весь интерфейс в тёмных тонах для комфортного использования
- **Responsive design** — адаптация под мобильные устройства
